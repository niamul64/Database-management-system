*** Important note on compiling, debugging and enabling triggers

 

To compile and debug a trigger:

 

1) enter the text of the trigger code into a text file and save it it into the folder where you are working. To make sure the trigger executes properly, put a forward slash "/" on its own on the last line of the file containing the trigger code.

 

2) Run the code in the file with the command

 

@"filename"

 

from the SQLPLUS prompt, where "filename" is the name of the file in which you stored the code.

 

3) If the system comes back saying there was compilation errors, examine these by typing

 

show errors

 

from the SQLPLUS prompt.

 

You should then edit the text file to deal with the compilation errors in the trigger. When the trigger compiles without an error message it is then available to use. Use the link from earlier on this page to see information about triggers and examples of trigger code to help deal with trigger compilation errors.

 

Enabling and disabling triggers

 

Before triggers can be used they have to be enabled. You can do this for all triggers with the command:

 

alter all triggers enable;

 

Similarly if you wish to switch all triggers off, issue the command

 

alter all triggers disable;

 

Triggers can be enabled or disabled for a specific table. e.g. to enable all triggers on the emp table:

 

alter all triggers on emp enable;
